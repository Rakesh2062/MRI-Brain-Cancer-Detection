==============================================================
VAIDHYANETRA AI - PROJECT ARCHITECTURE & FILE DESCRIPTIONS
==============================================================

This document explains the exact purpose of every single Python file and folder in the VaidhyaNetra AI Backend. The architecture is modular, meaning each specific task is separated into its own file so the code is easy to read, test, and scale!

---------------------------------
1. ROOT DIRECTORY
---------------------------------

• main.py
  - The core entry point for the FastAPI server. 
  - It ties all the other files together. It defines the API routes (like `/predict` and `/patient/{patient_id}`) and routes incoming internet requests to the appropriate background services.

• batch_test.py
  - A utility script used by developers to automatically send hundreds of MRI images from a local folder to the AI model in one click to evaluate the model's accuracy.

• requirements.txt
  - A list of all the Python libraries (like fastapi, torch, cv2) required to make this backend run.

• .env
  - A hidden configuration file that safely stores passwords and secret API keys (like your Cloudinary credentials or MongoDB URL) so they don't get hardcoded directly into the script.

• HOW_TO_RUN.txt
  - Setup and execution instructions for running the backend and testing endpoints via command line.


---------------------------------
2. SERVICES / (The AI Pipeline)
---------------------------------
This is the "Brain" of the backend. When an image arrives at main.py, it is passed sequentially through these files like an assembly line:

• services/preprocessing.py
  - Prepares the raw user-uploaded image for the AI. It resizes the image to exactly 256x256 pixels, converts color channels, and normalizes the lighting/contrast so the AI model doesn't get confused.

• services/segmentation.py
  - The "Eyes" of the AI. It takes the preprocessed image, feeds it through the PyTorch U-Net model, and outputs a binary mask (a map showing exactly which pixels are tumor and which are healthy brain).

• services/feature_extraction.py
  - Analyzes the binary mask generated by the segmentation model. It calculates the physical size of the tumor (in mm²) by counting pixels, and categorizes the severity level (Low, Moderate, High).

• services/confidence.py
  - Computes a mathematical probability score indicating how confident the AI is in its primary prediction.

• services/explainability.py
  - Responsible for interpretability. It generates the "Heatmap" or "Grad-CAM" overlay image, placing brightly colored red/yellow zones over the patient's MRI to visually show the doctor exactly *where* the AI was looking when it made its decision.

• services/uncertainty.py
  - Calculates an uncertainty factor to detect edge cases. If the AI is highly "uncertain", it raises a flag suggesting that a human doctor should manually review this specific MRI.

• services/growth_tracker.py
  - The longitudinal tracking tool. It looks up the patient's past historic scans in the database, compares the old tumor size to the current tumor size, and calculates the rate of tumor growth or shrinkage.

• services/recommendation.py
  - The clinical logic engine. Based on the tumor size, severity, and growth trend, it generates dynamic human-readable text recommending next steps (e.g., "Urgent surgery consult required" vs "Standard 12-month follow up").

• services/report_generator.py
  - The reporting engine. It gathers all the data from the services above (Heatmaps, URLs, Recommendations, Patient ID) and uses the `reportlab` library to render a professional, downloadable PDF file containing all the scan results.


---------------------------------
3. MODELS / (The AI Brain)
---------------------------------

• models/unet.py
  - Contains the pure mathematical PyTorch Class definition of the Neural Network architecture (The U-Net). This defines the structure of the layers that the AI uses to process the image.

• models/unet_model.pth
  - The "Memories" or "Weights" of the AI. This is a massive binary file containing hundreds of megabytes of trained mathematical parameters that tell the U-Net *how* to find brain tumors.


---------------------------------
4. DATABASE /
---------------------------------

• database/db_handler.py
  - The only file allowed to talk to MongoDB. It abstracts away all the complex queries. If you ever want to switch from MongoDB to PostgreSQL or MySQL in the future, you only need to change this one file.


---------------------------------
5. UTILS /
---------------------------------

• utils/cloudinary_helper.py
  - A small utility that handles secure file transfers. It takes the original MRI images and AI-generated heatmaps, securely uploads them to Cloudinary cloud storage, and fetches the permanent public URLs so the Front-End (Next.js) website can display them on the dashboard.
